# Banana Claims Configuration

Banana Claims uses two server configuration files and one claim-data file.

## Main Configuration

Path:

```text
config/bananaclaims.json
```

Default:

```json
{
  "configVersion": 1,
  "permissions": {
    "fabricPermissionsApiEnabled": true,
    "publicFallbackLevel": 0,
    "managementFallbackLevel": 0,
    "adminFallbackLevel": 3,
    "fallbackLevelOverrides": {}
  },
  "protection": {
    "denialMessagesEnabled": true,
    "denialMessageCooldownTicks": 20
  }
}
```

### Permission settings

- `fabricPermissionsApiEnabled`: Uses Fabric Permissions API and LuckPerms when available.
- `publicFallbackLevel`: Vanilla command level for public commands when an external provider is unavailable or disabled.
- `managementFallbackLevel`: Fallback for claim-management commands.
- `adminFallbackLevel`: Fallback for administration and protection bypass.
- `fallbackLevelOverrides`: Exact node-to-level overrides.

Values are clamped to vanilla levels `0` through `4`.

### Protection messages

- `denialMessagesEnabled`: Controls chat feedback; protection still applies when disabled.
- `denialMessageCooldownTicks`: Minimum delay between repeated denial messages for the same player and action.

Reload with:

```text
/claim admin reload config
```

## Preview Configuration

Path:

```text
config/bananaclaims-preview.json
```

Important groups:

- General duration, view range, glow color, and shadow settings
- Terrain-following border material and dimensions
- Corner anchor and full-height column settings
- Guide column material, spacing, and width
- Log and leaf filtering
- Fade animation
- Pulse animation

A complete reference file is included in the source package. Preview changes can be loaded without restarting:

```text
/claim admin reload preview
```

Existing active previews retain their current snapshot; newly created previews use the reloaded values.

## Claim Data

Path:

```text
config/bananaclaims/claims.json
```

Do not edit this file while the server is running. Banana Claims writes it through a temporary file and atomically replaces the live file when the operating system supports atomic moves.

Reload manually edited claim data with:

```text
/claim admin reload claims
```

An invalid reload leaves the current in-memory claims active. If the claim file is invalid during startup, Banana Claims stops initialization rather than loading an empty claim set that could overwrite recoverable data.

## Full Reload

```text
/claim admin reload
```

This reloads the main config, claims, and preview config.
